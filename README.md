## Таблица атрибутов и настроек JSON

| Атрибут / настройка | Что делает | Пример |
|---------------------|------------|--------|
| `[JsonPropertyName("name")]` | Задаёт точное имя поля в JSON | Вместо `"Name"` в JSON будет `"name"` |
| `[JsonIgnore]` | Поле не попадает в JSON  | Пароли, внутренние поля|
| `PropertyNamingPolicy.CamelCase` | Все поля в camelCase | `RealName` → `"realName"` |
| `JsonStringEnumConverter` | enum как строка | `"Marvel"` вместо `0` |
| `WriteIndented = true` | JSON с отступами  | Удобно при разработке |

## Главные выводы

### 1. Что такое сериализация и десериализация?

**Сериализация** — это процесс преобразования объекта C# (например, экземпляра класса `Hero`) в строку формата JSON.  
**Десериализация** — обратный процесс: из JSON-строки восстанавливается объект C#.

### 2. Зачем нужен `[JsonIgnore]`? Приведите реальный пример.

`[JsonIgnore]` исключает свойство из сериализации. 
- **Служебных данных**, которые не должны попадать клиенту (например, внутренние заметки разработчика, отладочная информация).
- **Конфиденциальных полей**, таких как пароль, хеш, токен доступа.

### 3. Почему enum лучше сериализовывать строкой, а не числом?

При сериализации enum числом (0, 1, 2) клиент получает непонятное значение. Ему нужно знать соответствие: 0 = Marvel, 1 = DC. Это неудобно, ведёт к ошибкам и жёсткой привязке к порядку значений.

### 4. Что изменилось в ответе после настройки `CamelCase`?

До применения `PropertyNamingPolicy.CamelCase` имена полей в JSON соответствовали именам свойств C# (с большой буквы):  
`"Id"`, `"RealName"`, `"PowerLevel"`.

После настройки все поля стали в **camelCase** (первая буква строчная):  
`"id"`, `"realName"`, `"powerLevel"`.
